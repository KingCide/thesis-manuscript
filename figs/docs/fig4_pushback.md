# 图 4-pushback：逐跳推回缓解传播示意图

## 一、设计目标

这张图是理解 CoopSatShield **推回缓解机制** 的核心示意图，需要展示：
1. **攻击流路径**：从 bot 接入点经多跳 ISL 到达受害链路
2. **PT 令牌反向传播**：从受害链路节点沿贡献最大的上游方向逐跳扩散
3. **逐跳缓解安装**：每个收到 PT 的节点安装软状态限速/降权规则
4. **推回效果**：攻击流在上游被提前丢弃/限速，释放中间链路带宽
5. **关键参数可视化**：R↑ 推回半径、r 分支数、T_life 生命周期

---

## 二、场景设定

### 拓扑（简化的 LEO 4-ISL 网格）

选取 12~15 颗卫星组成的局部网络拓扑，足以展示 R↑=3 的推回传播深度。

### 攻击设定

- **受害链路 e***：节点 V 与节点 V' 之间的 ISL
- **攻击路径**：2~3 条来自不同方向的攻击路径汇聚到 e*
- **bot 接入**：攻击流从地面 bot 通过上下行链路进入卫星网络
- **良性流量**：与攻击流共享链路的良性业务

## 四、视觉设计要素

### 节点表示

| 节点类型 | 形状 | 颜色 | 标注 |
|---------|------|------|------|
| 受害节点 V | 大圆 + 红色外环 | 红色边框 #D32F2F | "拥塞检测 c=1" |
| 安装 PT 的节点 | 圆 + 金色 shield 图标 | 金色边框 #F57C00 | "rank(a)=High" |
| 未影响节点 | 普通圆 | 灰色 | - |
| Bot 接入点 | 三角/菱形 | 暗红 | "Bot" |

### 链路表示

| 链路类型 | 样式 | 颜色 |
|---------|------|------|
| 受害链路 e* | 粗实线 + 红色闪烁标记 | 红色 #D32F2F |
| 攻击流经过的链路 | 中粗线 + 攻击流箭头叠加 | 灰色带红色箭头 |
| 推回后释放的链路 | 中粗线 + 绿色渐变 | 从红色渐变为绿色 |
| 普通 ISL | 细线 | 浅灰 |

### 箭头类型

| 箭头 | 样式 | 说明 |
|------|------|------|
| 攻击流方向 | 灰色半透明粗箭头（向下 ↓） | 从 bot 到 e* |
| PT 传播方向 | 金/橙色实箭头（向上 ↑） | 从 V 到上游 |
| 良性流恢复 | 绿色细箭头 | 推回后释放的带宽 |

### 推回半径标注

- 在 V 周围用**虚线弧**标出 R↑=1, R↑=2, R↑=3 的同心环
- 每层环旁标注 "R↑=1 hop", "R↑=2 hops", "R↑=3 hops"
- 环内的节点显示已安装缓解规则

### 入端口贡献分布可视化

在某个关键节点（如 U₁）旁边展示一个小型**饼图或条形图**：

```
U₁ 节点入端口贡献:
  p₁ (来自 U₂): 70% ████████████████████
  p₂ (来自 N): 20% ██████
  p₃ (来自 N'): 10% ███
  → 选择 p₁ (r=1 时) 或 p₁+p₂ (r=2 时) 继续推回
```

### 软状态生命周期时间线

在图右侧或下方附加一个小型时间线：

```
epoch τ:    PT 生成 + 安装
epoch τ+1:  规则生效，攻击流被限速
  ...
epoch τ+10: T_life 过期，规则自动清除
epoch τ+11: 若攻击仍在，新 PT 刷新规则
```

## 八、绘制优先级

推荐以 **阶段 2 的网络拓扑 + PT 传播** 为主体，占图的 70% 面积。阶段 1（拥塞检测）和阶段 3（效果对比）分别作为左上角和右下角的辅助标注框，占 15% 各。

核心视觉效果：读者应能**一眼看到** PT 令牌从红色受害链路处"逆流而上"向 bot 方向扩散，沿途节点逐个亮起金色 shield 标记，攻击流在上游被截断，中间链路从红色变为绿色。
