# 图 3-2：F1 对比图（退化攻击下的检测性能）

## 一、原理（为什么需要这张图）

### 论文中的理论基础

当攻击者采用退化策略（多 decoy 分摊）时，总攻击量不变但分散到 M 个目的 key 上，导致每个 key 的聚合速率降低。这张图验证：仅凭聚合速率（rate-only）是否足以检测？引入 FO/FI 后能否恢复检测能力？

### 退化攻击模型（表 3-3）

固定参数：总攻击量 A=10Gb/s，B=2000 个 bot，r=5Mb/s/bot。当 M 增大时：
- 每个 decoy 聚合速率 = A/M
- M=1 时：10 Gb/s（远超良性 p99）
- M=1000 时：10 Mb/s（混入良性长尾，rate-only 失效）

### 两种检测方案（S0 vs S1）

| 方案 | 公式 | 含义 |
|------|------|------|
| S0 (rate-only) | S(a) = R&#x0303;(a) | 仅归一化速率 |
| S1 (+FO/FI) | S(a) = 0.5R&#x0303; + 0.5F&#x0303; | 加入 fan-in/fan-out 特征 |

> **注意**：此图不包含 S2 (+persist)。原因见"设计决策"一节。

### FO/FI 的检测原理

即使攻击速率被分摊到低于良性 p99，攻击 decoy 仍有异常多的不同源地址。因为 B=2000 个独立 bot 分配给 M 个 decoy，每个 decoy 的 FI = B/M。当 M=100 时 FI=20，仍远高于良性 key 的典型 FI（1~5）。

### 位置

这是第三章的核心实验图，位于"评价指标与绘制方式"节，论证 FO/FI 多签名方案在静态退化场景下的必要性和有效性。

## 二、设计决策：为什么 fig3_2(b) 不含 S2

### 问题

在 2026-02-26 的修改前版本中，fig3_2(b) 包含 S0/S1/S2 三条曲线。但数据显示 S1 和 S2 在全 M 范围几乎完全重叠：

| M | S0 | S1 | S2 | S2-S1 差值 |
|---|---|---|---|---|
| 100 | 0.017 | 0.908 | 0.909 | +0.001 |
| 500 | 0.042 | 0.426 | 0.445 | +0.019 |
| 1000 | 0.041 | 0.420 | 0.460 | +0.040 |

### 根因分析

fig3_2 的退化攻击是**静态场景**（攻击在所有 epoch 中持续不变）。在这种条件下：

1. **攻击 key**：速率抖动极小（0.8~1.2 倍），每个 epoch 都进入 top-K → `P̃ = 5/5 = 1.0`
2. **良性重流 key**（~20 个，rate=290~606 Mb/s）：也几乎每个 epoch 都在 top-K 里 → `P̃ ≈ 1.0`
3. **结论**：P̃ 对攻击 key 和 FP key 都饱和到 1.0，无法提供区分能力

S2 相比 S1 的权重变化从 `(0.5, 0.5, 0)` → `(0.5, 0.5, 0.2)`——persistence 是额外加分。
但在静态场景中，持续出现的攻击 key 和良性重流 key 都获得 `P̃ ≈ 1.0`，额外加分 `0.2 × 1.0 = 0.2` 对两者影响相同，无法改善 F1。

### 解决方案

- **fig3_2(b)**（本图）：仅展示 S0 vs S1，聚焦于 FO/FI 在静态退化场景下的核心贡献
- **fig3_4**（on-off 时序图）：展示 S1 vs S2，聚焦于 persistence 在动态场景下的核心贡献
- **论文叙事**：FO/FI 是静态退化的解药，persistence 是动态退化的解药，两者各有分工

## 三、预期结论（图应该呈现什么趋势）

### 图 3-2(a)：rate-only 检测器各指标随 M 变化

- **Top-K 候选 Recall**：M≤100 时接近 1.0，M≥500 时降至 ~0.3（攻击 key 速率不足以进入 top-K）
- **Rate-only Recall**：M≤10 时为 1.0，M=50 骤降（论文第 555 行："rate-only 的 Recall 先下降"）
- **Rate-only Precision**：全程低，因为 ~20 个良性重流 key 始终被误判
- **Rate-only F1**：全程低于 0.5，说明仅靠速率无法有效检测退化攻击

### 图 3-2(b)：S0 vs S1 F1 对比 —— 倒 U 型趋势分析

S1 的 F1 呈现 **先升后降的倒 U 型**，峰值在 M=100 附近。

#### 三阶段行为

1. **重叠段（M≤10）**：S0 ≈ S1，两条线几乎重叠。此区间攻击 key 速率极高（≥1 Gb/s），rate-only 已能检出全部攻击 key（Recall=1.0）。额外的 FO/FI 特征不提供增益，反而因良性重流在多维度上也表现突出而略增 FP。

2. **分离段（M=50~100）**：S1 大幅超越 S0。攻击 per-key 速率降至 100~200 Mb/s，rate-only 失效（R̃ 接近良性），但 FO/FI 仍可区分（FI=20~40 远超良性 FI=1~5）。这是多签名的核心价值区间。

3. **衰减段（M≥500）**：S1 F1 下降。攻击 per-key 速率降至 20~35 Mb/s，部分攻击 key 落入良性速率区间而脱离 top-K 候选集（TopK_Recall~0.3）。

#### ~20 个良性重流 FP 的来源

FP 数量（~20）在所有 M 值下近似恒定，来源是真实的良性重流 key（如热门目的地）：rate=290~606 Mb/s，FI=13~26，score 天然 > tau=0.5。这是任何基于 rate+FI 的检测器的固有下限。在实际部署中，队列缓解是渐进式（非二元阻断），~20 个被轻微降级的良性 key 是可接受的代价。

## 七、修改历史

### 2026-02-26：移除 S2、重新定位 persistence 展示场景

**问题**：S1 和 S2 在全 M 范围几乎完全重叠（最大差值仅 +0.04）。

**根因**：fig3_2 是静态攻击场景，攻击 key 和良性重流 key 的 P̃ 均饱和到 ~1.0，persistence 无法提供区分能力。从 S1(0.5,0.5,0) 到 S2(0.4,0.4,0.2) 只是将 R̃+F̃ 的有效权重削弱了 0.2，换来一个对所有候选 key 都相同的常数偏移。

**修复**：
1. fig3_2(b) 移除 S2 曲线，仅保留 S0 vs S1
2. Persistence 的核心价值转移到 fig3_4（on-off 动态场景）展示
3. 论文叙事调整：FO/FI 对应静态退化，persistence 对应动态退化

### 2026-02-26：去掉噪声 key + K 自适应（P1-1）

**问题**：S1 F1 呈倒 U 型，M=500 从 0.85 骤降至 0.14（含噪声时）。

**根因**：5000 噪声 key vs 377 真实 key → 噪声主导 top-K。

**修复**：
1. 检测实验中 `n_noise_keys=0`
2. K 自适应：`K = clip(0.3 * N_keys, 50, 1000)`

### 2026-02-25：修复 Recall 恒为 1.0 / F1 趋近 0 的问题（P0-2）

**问题**：K=10000 >> 总 key 数，top-K 返回全集；fig3_2(a) 无阈值判决。

**修复**：全局 K 从 10000 改为 1000；fig3_2(a) 改用完整管线。
