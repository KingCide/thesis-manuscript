# 图 4-arch：CoopSatShield 体系结构图

## 一、设计目标

这张图是第四章的**核心方法图**，展示 CoopSatShield 系统的完整架构。需要同时表达：
1. **单节点内部结构**：P4 可编程交换机（数据面）+ 星上 Agent（控制面）的分工
2. **多节点协同机制**：EG/PT 两类消息在 ISL 上的传播
3. **三层逻辑**：本地证据抽取 → 抑制型扩散协同 → 推回缓解执行
4. **与第三章的衔接**：表明本地检测模块直接复用 MS-SatShield

#### 数据面模块说明

| 模块 | 功能 | 来源 |
|------|------|------|
| top-K 候选生成 | 以 (key, len) 更新计数，维护 H_t | 第三章 MS-SatShield |
| FO/FI 位图 | 仅对 H_t 中候选 key 维护 bitmap(a) | 第三章 MS-SatShield |
| 入端口贡献统计 | 维护 bytes(a,p)，用于 pushback 路径选择 | **第四章新增** |
| 队列调度 + Pushback 规则 | rank(a)→队列映射 + PT 安装的软状态限速 | 第三章基础 + **第四章扩展** |
| EG/PT 消息解析 | 轻量 header 解析 + TTL 递减 | **第四章新增** |
| seen-set | Bloom Filter 去重 EG 消息 | **第四章新增** |

#### 控制面模块说明

| 模块 | 功能 | 触发条件 |
|------|------|---------|
| 多签名评分 | 从数据面读取 R(a), F(a), P(a)，计算 S_v,τ(a) | 每 epoch |
| 证据缓存 E_{v,τ} | 存储本地和收到的 EG 证据摘要 | 收到 EG 时 |
| 抑制型扩散决策 | 监听等价证据 cnt，决定发送/抑制 | c(v,e)=1 或收到新颖 EG |
| 协同聚合 S^net | 加权聚合 ROI 内评分 | EG 汇聚后 |
| Pushback 判定 | S^net≥Θ_pb 时生成 PT 令牌 | 协同评分完成后 |
| EG 消息处理/合并 | top-n 合并 + digest 并集 | 收到 EG 时 |

### 右栏：多节点协同拓扑（网络视角）

展示 4~6 颗卫星的网格拓扑（模拟 Starlink Shell 1 的 4-ISL 连接），标注两类消息的传播。

```
                    ┌───┐
          EG(TTL=2) │ D │
            ↗       └─┬─┘
      ┌───┐          │
      │ C │──────────┘
      └─┬─┘
        │ EG(TTL=3)
  ┌───┐ ↑
  │ B │─┘           受害链路 e*
  └─┬─┘        ═══════════════
    │              ↕
  ┌─┴─┐          ┌───┐
  │ A │──────────│ V │  ← 受害节点，触发 EG
  └───┘          └─┬─┘
    ↑               │ PT(R↑=3)
    │ PT(R↑=2)      ↓
  ┌───┐          ┌───┐
  │ E │          │ F │
  └───┘          └───┘
    ↑ PT(R↑=1)
  ┌───┐
  │ G │ ← 攻击流入口（bot 接入）
  └───┘
```

#### 右栏标注要素

1. **受害链路 e***：用红色粗线标注，节点 V 检测到拥塞
2. **EG 消息传播方向**：从 V 向外扩散（蓝色箭头），标注 TTL 递减
3. **ROI 边界**：虚线圆/椭圆标注 R=3 的 ROI 范围
4. **PT 消息传播方向**：从 V 向上游反方向传播（橙色/金色箭头），标注 R↑ 递减
5. **攻击流路径**：灰色半透明箭头标注攻击流从 bot(G) → E → A → V → e* 的路径
6. **抑制标注**：某个节点旁标注"🚫 已听到 k=3 条等价证据，抑制发送"
7. **良性流恢复**：e* 链路处标注攻击流被丢弃/限速，良性流通过

## 四、与第三章 fig3-5 架构图的关系

第三章 fig3-5 展示了 **单颗卫星内部** 的 MS-SatShield 架构：
- 数据面：top-K → FO/FI bitmap → 队列调度
- 控制面：epoch 读取 → 多签名评分 → rank 写回

第四章图在此基础上：
1. **数据面新增**：入端口贡献统计 π(a,p)、EG/PT 解析 + TTL 递减、seen-set 去重、Pushback 软状态规则
2. **控制面新增**：证据缓存、抑制型扩散决策、协同聚合 S^net、Pushback 判定、EG 合并
3. **网络视角新增**：多节点拓扑 + EG/PT 消息传播

建议在图中用 **★ 或不同边框颜色** 标注"第四章新增"模块，使读者一眼看出本章的增量贡献。

## 五、绘制建议

### 尺寸与比例

- figsize 建议 (14, 8) 或 (16, 9)
- 左栏占 55%，右栏占 45%
- 单节点详图要足够大以容纳所有模块标签

### 数据流标注（左栏）

在左栏节点图中标注关键数据流箭头：

1. **包处理流**（数据面内部，从左到右）：
   `入端口 p → 解析 → top-K 更新 → [候选命中?] → FO/FI 更新 → rank 查询 → 队列入队 → 出端口`

2. **epoch 控制流**（控制面 ↔ 数据面，垂直）：
   `Agent 读取 top-K / bytes / bitmap → 计算 S(a), rank → 写回交换机`

3. **协同消息流**（控制面 ↔ ISL）：
   `收到 EG → 去重(seen-set) → 相关性↷过滤(Ω≥θ) → 缓存合并 → 抑制决策 → [发送/抑制]`
   `S^net≥Θ_pb → 生成 PT → 查 π(a,p) → 选上游端口 → 发送 PT`

4. **PT 执行流**（ISL → 数据面）：
   `收到 PT → 安装软状态规则 rank(a) → TTL 倒计时清理`

### 标注文字

- 模块名用**粗体**
- 公式变量用数学斜体（如 $S_{v,\tau}(a)$, $\pi(a,p)$）
- 在连接箭头旁标注说明（如 "每 epoch 读取", "仅候选集"）
- 第三章复用部分标注 "⟵ Ch.3 MS-SatShield"

### 关键标注节点

在右栏网络拓扑中，标注以下关键信息：
- ROI 半径 R=4（虚线圆形区域）
- 推回半径 R↑=3（沿攻击路径反方向的标注）
- 抑制机制效果（某节点旁标注 "suppress: cnt≥k"）
- EG 消息 TTL 递减过程
