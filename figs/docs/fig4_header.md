# 图 4-header：EG/PT 消息 Header 格式图

## 一、设计目标

这张图以 **bit-level 字段布局** 的形式，展示 CoopSatShield 定义的两类星间控制消息：
1. **Evidence Gossip (EG)** — 用于证据传播与聚合
2. **Pushback Token (PT)** — 用于缓解推回

图的核心目的是让读者一眼看出消息结构的**紧凑性**（总大小在 KB 级）和**可编程数据面友好性**（常数时间解析）。

---

## 二、EG 消息字段定义（来自表 4-1 和正文）

### 字段布局（从高位到低位）

```
┌───────┬──────────┬─────┬─────────┬──────────────┬────────────────────────────────┬──────────┐
│ type  │  epoch   │ ttl │  nonce  │   link_id    │           digest               │cong_level│
│ 4b    │  16b     │ 4b  │  32b    │   32b        │          128b (BF)             │   8b     │
├───────┴──────────┴─────┴─────────┴──────────────┴──────────────────────────────────┴──────────┤
│                                    固定头部 224 bits (28 bytes)                                │
├─────────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                               │
│  ┌──────────────────────────────────────────────────────────────────────────────────────────┐  │
│  │  top-n 可疑 key 列表 (可变长度)                                                           │  │
│  │                                                                                          │  │
│  │  ┌──────────┬──────────┐  ┌──────────┬──────────┐       ┌──────────┬──────────┐          │  │
│  │  │ â_1 (32b)│ ŝ_1 (8b) │  │ â_2 (32b)│ ŝ_2 (8b) │  ...  │ â_n (32b)│ ŝ_n (8b) │          │  │
│  │  └──────────┴──────────┘  └──────────┴──────────┘       └──────────┴──────────┘          │  │
│  │                                                                                          │  │
│  │  每个 entry: 40 bits (5 bytes) × n 个                                                    │  │
│  └──────────────────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                               │
├─────────────────────────────────────────────────────────────────────────────────────────────────┤
│  总大小 = 28 + 5n bytes                                                                       │
│  n=10 → 78 bytes; n=50 → 278 bytes; n=100 → 528 bytes                                       │
└─────────────────────────────────────────────────────────────────────────────────────────────────┘
```

### 字段说明

| 字段 | 位宽 | 说明 |
|------|------|------|
| `type` | 4 bit | 消息类型标识：0x1=EG, 0x2=PT |
| `epoch` | 16 bit | epoch 编号，用于软状态一致性与过期清理。$2^{16}=65536$ epoch，足够周期循环 |
| `ttl` | 4 bit | 剩余传播跳数，最大值 15（实际 R 推荐为 4，上限足够） |
| `nonce` | 32 bit | 消息唯一标识，用于 seen-set 去重（Bloom Filter 中 hash 此值） |
| `link_id` | 32 bit | 触发拥塞的链路标识（源节点 ID 16b + 出端口 4b + 保留 12b） |
| `digest` | 128 bit | 候选 key 集合的 Bloom Filter 摘要。接收端以 O(1) 判断本地候选是否重叠 |
| `cong_level` | 8 bit | 拥塞强度量化（0-255），来自数据面 traffic manager 的丢包率/队列占用 |
| `â_i` | 32 bit | 第 $i$ 个可疑 key 的短哈希（CRC32 或 xxHash32） |
| `ŝ_i` | 8 bit | 第 $i$ 个可疑 key 的量化评分（0-255，线性映射 $[0,1]$） |

### 设计理由

1. **固定头部 28 字节**：可在 P4 交换机的 parser 中以固定偏移解析（满足线速约束）
2. **digest 用 128-bit Bloom Filter**：支持 O(1) 判断重叠度 $\Omega(M,v)$，无需逐条比对
3. **â_i 用 32-bit 短哈希**：在实际 key 空间（src_prefix + dst_prefix）中碰撞率极低，且节省带宽
4. **ŝ_i 用 8-bit 量化**：分辨率 1/256 足够排序和阈值判断，避免浮点传输

## 三、PT 消息字段定义（来自算法 2 和正文）

### 字段布局

```
┌───────┬──────────┬──────┬──────────┬─────────────┬──────┬───────────┐
│ type  │  epoch   │ R_up │  key_id  │    rank     │T_life│  reserved │
│ 4b    │  16b     │ 4b   │  32b     │   8b        │ 8b   │   &nbsp;  │
├───────┴──────────┴──────┴──────────┴─────────────┴──────┴───────────┤
│                       固定大小 72 bits (9 bytes)                      │
└─────────────────────────────────────────────────────────────────────┘
```

### PT 关键字段

| 字段 | 位宽 | 说明 |
|------|------|------|
| `type` | 4 bit | 0x2 = PT |
| `epoch` | 16 bit | 生成 PT 时的 epoch 编号 |
| `R_up` | 4 bit | 剩余推回跳数，初始值为 $R_\uparrow$（推荐 3），每跳递减 1 |
| `key_id` | 32 bit | 目标攻击 key 的短哈希 $\hat{a}$（与 EG 中 â_i 相同编码） |
| `rank` | 8 bit | 缓解优先级，控制队列降权/限速强度 |
| `T_life` | 8 bit | 软状态生存期（以 epoch 为单位），过期后数据面自动清除规则 |

### 设计理由

1. **PT 极小（9 字节）**：推回令牌设计为最小化开销，仅携带执行缓解所需最少信息
2. **R_up 用 4 bit**：最大推回跳数 15，实际推荐 R↑=3 足够
3. **T_life 用 8 bit**：最大生存 255 epoch，防止拓扑变化后产生"幽灵规则"
4. **rank 用 8 bit**：与数据面队列优先级映射对齐（通常 8 级队列足够）

## 四、图像视觉设计

### 布局

上下两栏：EG 消息（上方，主体较大）和 PT 消息（下方，精简），中间用细分隔线。

```
┌────────────────────────────────────────────────────────┐
│  ┌──────────────────────────────────────────────────┐  │
│  │  Evidence Gossip (EG) - 28+5n bytes              │  │
│  │                                                  │  │
│  │  ┌────┬───────┬───┬──────┬───────┬───────┬─────┐│  │
│  │  │type│ epoch │ttl│nonce │link_id│digest │cong_││  │
│  │  │ 4b │ 16b   │4b │ 32b  │ 32b   │ 128b  │ 8b  ││  │
│  │  └────┴───────┴───┴──────┴───────┴───────┴─────┘│  │
│  │  ┌────────┬──────┐   ×n entries                  │  │
│  │  │ â_i 32b│ŝ_i 8b│                               │  │
│  │  └────────┴──────┘                               │  │
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  ┌──────────────────────────────────────────────────┐  │
│  │  Pushback Token (PT) - 9 bytes                   │  │
│  │                                                  │  │
│  │  ┌────┬───────┬────┬───────┬────┬──────┐         │  │
│  │  │type│ epoch │R_up│key_id │rank│T_life│         │  │
│  │  │ 4b │ 16b   │ 4b │ 32b   │ 8b │  8b  │         │  │
│  │  └────┴───────┴────┴───────┴────┴──────┘         │  │
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  ┌──────────────────────────────────────────────────┐  │
│  │  ISL 传播路径示意                                  │  │
│  │  EG: V → (TTL=R) → ... → (TTL=0, 停止)          │  │
│  │  PT: V → 上游 ←(R↑) → ... → (R↑=0, 停止)       │  │
│  └──────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────┘
```

### 视觉风格

采用 **RFC 风格** 的 bit-level 字段图：

1. 每个字段用**矩形格子**表示，宽度按 bit 数等比例缩放
2. 上方标注字段名，下方标注位宽
3. 不同类型字段用不同颜色填充：
   - **控制字段**（type/epoch/ttl/R_up）: 浅蓝 #E3F2FD
   - **标识字段**（nonce/link_id/key_id）: 浅黄 #FFF9C4
   - **证据字段**（digest/â/ŝ/cong_level）: 浅绿 #E8F5E9
   - **缓解字段**（rank/T_life）: 浅橙 #FFF3E0
4. EG 和 PT 用不同底色区分（EG 蓝色调，PT 橙色调）
5. 可变长部分（top-n key 列表）用虚线边框 + "×n" 标注

### 大小总结标注

在图底标注尺寸对比：


## 五、下方辅助：ISL 消息传播路径示意

在字段图下方，用一排简化的卫星节点 + 箭头展示消息沿 ISL 传播的过程：

**EG 传播**:
```
[V]  ──EG(TTL=4)──> [N₁] ──EG(TTL=3)──> [N₂] ──EG(TTL=2)──> [N₃]  ... ──(TTL=0, 停止)
 │                    │                    │
 └─ 拥塞触发           └─ digest 重叠检查     └─ 抑制: cnt≥k, suppress
```

**PT 传播**:
```
[V]  ──PT(R↑=3)──> [U₁] ──PT(R↑=2)──> [U₂] ──PT(R↑=1)──> [U₃]  (R↑=0, 停止)
                    │                    │                    │
                安装 rank(a)          安装 rank(a)         安装 rank(a)
                T_life=10ep          T_life=10ep           T_life=10ep
```
